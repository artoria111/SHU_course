## 一、系统概述

智能招聘匹配系统是一个基于 NLP 技术的职位推荐问答平台。用户通过自然语言描述自身技能、经验和求职意向，系统自动识别关键实体、构建求职者画像，利用语义搜索和加权匹配算法推荐最合适的职位，并提供详细的岗位职责、任职要求和面试流程。

**应用场景**：求职者快速匹配职位、了解岗位要求、评估薪资水平。

**核心价值**：将传统招聘网站的"关键词搜索+筛选"模式升级为"自然语言描述+智能匹配"的对话式体验。

## 三、详细设计

### 3.1 功能模块

| 序号 | 功能 | 描述 | NLP相关 |
|------|------|------|---------|
| 1 | 技能实体识别 | 从用户输入中提取技能关键词，支持中英文混排 | ✅ NER |
| 2 | 意图分类 | 识别用户意图（求职咨询/技能评估/薪资咨询/职位查询/闲聊） | ✅ 文本分类 |
| 3 | 技能同义词标准化 | 将缩写映射为标准名（js→JavaScript, k8s→Kubernetes） | ✅ 实体规范化 |
| 4 | 语义职位搜索 | 将技能描述编码为向量，与职位库做余弦相似度检索 | ✅ 语义搜索 |
| 5 | 智能匹配打分 | 三层加权：技能重叠 + 需求匹配 + 职位匹配 | ✅ 文本匹配 |
| 6 | 用户画像抽取 | 正则提取工作经验、学历、期望薪资、意向城市 | ✅ 规则抽取 |
| 7 | 职位详情推荐 | 展示匹配职位的岗位职责、任职要求、福利待遇、面试流程 | ❌ 信息展示 |
| 8 | 多轮对话管理 | 会话级技能标签累积、画像渐进补全 | ❌ 会话管理 |

### 3.2 NLP 技术实现

**技术1：命名实体识别（NER）**  
- 方案：双层匹配（500+技能关键词库 + spaCy 词性标注辅助）  
- 代码位置：`extract_skills_and_positions()`  
- 示例：输入"我会Java和Spring Boot" → 输出 ["Java", "Spring Boot"]

**技术2：意图分类**  
- 方案：正则模式匹配，5类意图（求职咨询/技能评估/薪资咨询/职位查询/闲聊）  
- 代码位置：`classify_intent()`  
- 规则定义：`INTENT_PATTERNS` 字典，每个意图关联多个正则模式

**技术3：实体规范化（同义词标准化）**  
- 方案：30+条缩写→标准名映射表  
- 代码位置：`normalize_skills()` + `SKILL_SYNONYMS`  
- 示例：js→JavaScript, vue→Vue.js, k8s→Kubernetes, llm→大语言模型

**技术4：语义搜索（向量匹配）**  
- 方案：SentenceTransformer (`paraphrase-multilingual-MiniLM-L12-v2`)  
- 代码位置：`match_positions()` → `util.semantic_search()`  
- 流程：用户技能文本 → 768维向量 → 与115个职位向量计算余弦相似度 → Top-K

**技术5：文本匹配与打分**  
- 方案：基于语义相似度的三层加权打分  
- 代码位置：`match_positions()` + `check_requirements()`  
- 权重：技能重叠 ×(1+0.2n) → 需求匹配 ×1.3 → 职位匹配 ×1.5

**技术6：规则信息抽取**  
- 方案：正则表达式 + 关键词列表  
- 代码位置：`extract_user_profile()`  
- 抽取字段：工作经验（6种模式）、学历、期望薪资（3种模式）、意向城市

**附加：依存句法分析**  
- 工具：spaCy `zh_core_web_sm`（含分词器、词性标注器、依存分析器）  
- 用途：辅助 NER 实体识别中的词性判断

### 3.3 数据说明

| 数据文件 | 来源 | 数量 | 字段 |
|----------|------|------|------|
| `position_features_info.json` | 猎聘爬虫 + 手工IT岗位 | 115个 | name, requirements, salary_range, skills[] |
| `position_detail_info.json` | 模板生成 + 爬虫填充 | 115个 | name, summary[], detail[].title+content |

**数据 Pipeline**：
```
scrape_positions.py           build_app_data.py            app_recruitment.py
 猎聘搜索页                   合并+去重                    加载JSON
 → 提取/zp链接                → 技能推断（词典）            → SentenceTransformer编码
 → 详情页解析求职卡片          → JD模板生成                 → 构建职位向量索引
 → scraped_positions.json     → position_*.json            → 在线匹配
```

爬虫数据覆盖 13 个城市、215 个技能关键词，合并 IT 手工岗位后确保技术类职位的匹配覆盖。

### 3.4 界面设计

- 左侧深紫渐变侧边栏：Logo、新建对话按钮、可滚动的会话列表（hover 显示删除按钮）
- 右侧主区域：顶栏（系统名称 + NLP驱动徽章） + 聊天区（空状态封面 / 消息气泡） + 输入区
- 消息气泡：用户消息（紫色渐变靠右）、AI消息（白色卡片靠左）
- 自定义确认弹窗：替代浏览器原生 confirm，圆角卡片+毛玻璃遮罩+滑入动画
- 响应式适配：`@media (max-width: 768px)` 移动端适配

## 六、总结

本系统以 NLP 技术为核心，实现了从自然语言求职描述到智能职位推荐的完整流程。系统涵盖 **6 项 NLP 技术**（命名实体识别、意图分类、实体规范化、语义搜索、文本匹配打分、规则信息抽取），提供 **8 个功能点**（其中 6 个 NLP 相关），满足课程大作业的全部要求。

系统采用模块化设计，数据层（爬虫→清洗→入库）、模型层（NER/语义搜索/规则引擎）、应用层（Flask Web/对话管理）清晰分离，便于扩展和维护。
